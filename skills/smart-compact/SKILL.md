# Smart Compact

Generate a structured, explicit summary of the current session and save it for the next one.

## When it triggers

- User says `/smart-compact`
- User asks to "save context" or "save the session"
- User wants to `/clear` but doesn't want to lose the work done

## Procedure

### Step 1: Analyze the session

Review the current conversation. Identify:
- **What was done** and **why** (motivation, trigger, user request)
- **Decisions made** with full reasoning
- **Problems encountered** and how they were resolved
- **Files modified** and nature of changes
- **What's left to do**

### Step 2: Gather data from files

Run:
```bash
git diff --stat 2>/dev/null
git diff --name-only 2>/dev/null
```

This confirms files touched. If not in a git repo, base everything on the conversation.

### Step 3: Determine the file path

The context file goes in:
```
~/.claude/projects/<PROJECT_PATH>/smart-compact-context.md
```

Where `<PROJECT_PATH>` is the current working directory with `/` replaced by `-` and the first `/` becomes `-`.
Example: `/Users/john/Projects/my-app` → `-Users-john-Projects-my-app`

Use the Bash tool to determine the path:
```bash
echo "$PWD" | sed 's|^/|-|' | sed 's|/|-|g' | sed 's| |-|g'
```

### Step 4: Read existing file (if present)

Use Read to check `smart-compact-context.md`. If it exists:
- Keep previous sessions (last 2-3)
- Update the "Project status" section
- Add a new session section
- If the file exceeds ~1500 words, prune oldest sessions keeping only decisions and lessons

If it doesn't exist, create from scratch.

### Step 5: Write the file

Use Write to create/update the file following this format:

```
# Smart Compact Context
> Generated by /smart-compact. Last updated: [DATE TIME]

## Project status
[Self-contained summary of current state. Must be understandable
without remembering the conversation.]

## Session [DATE TIME]

### Work done
- [What] — [Why, explicit motivation]

### Files modified
- `path/file` — [Description of change and motivation]

### Decisions made
[Only significant choices. For each:]
- **Decision:** [Full sentence]
  **Why:** [Complete reasoning, not abbreviated]
  **Alternatives considered:** [What was discarded and why]

### Problems and solutions
- **Problem:** [What happened]
  **Root cause:** [Why]
  **Solution:** [How it was resolved]
  **Lesson:** [What to avoid in the future]

### Next steps
- [ ] [What's left to do]
```

### Step 6: Apply the explicitness test

Before saving, for each entry ask yourself:
> "If I read this in 1 week without remembering the conversation, will I understand everything?"

If the answer is no, add detail. Prefer explicitness over brevity.

**Don't save:**
- Trivial changes (typos, formatting)
- Failed attempts without a useful lesson
- Technical details that can be re-read from code

**Always save:**
- Decisions with the why
- Non-obvious problems with the solution
- Context that isn't in the code (user motivations, external constraints)

### Step 7: Confirm

After writing the file, respond:

"Context saved to `smart-compact-context.md` ([size]). Run `/clear` when you want a fresh start — next session I'll ask if you want to load the context."

---

## Rules

1. **Don't read the file unless needed.** Only read it if it exists AND you're about to update it.
2. **The file is a living document.** It evolves, it's not overwritten from scratch.
3. **Maximum ~2000 tokens.** Prune old sessions if needed, keeping decisions and lessons.
4. **Don't auto-clear.** Suggest it, the user decides.
5. **Consistent format.** Always follow the structure defined above.
